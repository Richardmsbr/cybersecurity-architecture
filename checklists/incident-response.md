# Incident Response Readiness Checklist

Comprehensive checklist for assessing organizational incident response capabilities.

---

## Governance and Planning

### IR Program

- [ ] Incident response policy approved
- [ ] IR plan documented and current
- [ ] Executive sponsorship established
- [ ] Budget allocated for IR
- [ ] Regular program reviews conducted

### Roles and Responsibilities

- [ ] IR team members identified
- [ ] Roles clearly defined (RACI)
- [ ] Backup personnel assigned
- [ ] External resources identified (forensics, legal)
- [ ] Escalation paths documented

### Contact Information

- [ ] Internal contacts current
- [ ] External contacts current (vendors, law enforcement)
- [ ] After-hours contacts available
- [ ] Communication tree tested

---

## Preparation

### Documentation

- [ ] IR playbooks developed
- [ ] Playbooks cover major incident types
- [ ] Procedures are step-by-step
- [ ] Decision trees documented
- [ ] Documentation accessible during incidents

### Tools and Resources

- [ ] Forensic tools available
- [ ] Evidence collection kits ready
- [ ] Secure communication channels
- [ ] War room/bridge capability
- [ ] Backup investigation systems

### Training

- [ ] IR team trained
- [ ] Training records maintained
- [ ] Certifications current (GCIH, GCFA)
- [ ] Annual refresher training
- [ ] Cross-training for redundancy

---

## Detection Capabilities

### Monitoring

- [ ] SIEM deployed and tuned
- [ ] EDR on all endpoints
- [ ] Network monitoring active
- [ ] Log aggregation complete
- [ ] 24/7 monitoring coverage

### Alerting

- [ ] Alert thresholds defined
- [ ] Critical alerts immediate notification
- [ ] Alert routing configured
- [ ] False positive rate acceptable
- [ ] Alert acknowledgment tracked

### Detection Coverage

- [ ] MITRE ATT&CK coverage assessed
- [ ] Detection rules regularly updated
- [ ] Threat intelligence integrated
- [ ] Hunt operations conducted
- [ ] Detection gaps identified

---

## Response Capabilities

### Containment

- [ ] Network isolation capability
- [ ] Endpoint isolation capability
- [ ] Account disable procedures
- [ ] Service shutdown procedures
- [ ] Containment decision authority defined

### Eradication

- [ ] Malware removal procedures
- [ ] System rebuild procedures
- [ ] Credential reset procedures
- [ ] Persistence removal checklist
- [ ] Clean image availability

### Recovery

- [ ] Recovery procedures documented
- [ ] Backup restoration tested
- [ ] Recovery priorities defined
- [ ] Validation procedures
- [ ] Return to normal criteria

---

## Evidence and Forensics

### Evidence Collection

- [ ] Evidence collection procedures
- [ ] Chain of custody forms
- [ ] Evidence storage (secure)
- [ ] Forensic imaging capability
- [ ] Memory acquisition tools

### Forensic Analysis

- [ ] Forensic analysis capability
- [ ] Malware analysis capability
- [ ] Timeline analysis tools
- [ ] Log analysis tools
- [ ] External forensics contacts

### Documentation

- [ ] Incident documentation templates
- [ ] Evidence logs maintained
- [ ] Actions timestamped
- [ ] Analyst notes captured
- [ ] Screenshots/recordings captured

## Communication

- [ ] Executive briefing templates
- [ ] Status update templates
- [ ] All-hands communication plan

### External Communication

- [ ] Media response plan
- [ ] Customer notification templates

- [ ] Law enforcement engagement procedures

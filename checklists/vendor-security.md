# Vendor Security Assessment Checklist

Comprehensive checklist for evaluating third-party vendor security posture.

---

## Company and Governance

### Company Information

- [ ] Company legal name and registration
- [ ] Physical address verified
- [ ] Years in business
- [ ] Financial stability assessed
- [ ] References checked
- [ ] Public security incidents researched

### Security Governance

- [ ] Security policy exists
- [ ] Security officer/team identified
- [ ] Board/executive oversight
- [ ] Security budget allocated
- [ ] Security roadmap exists

### Compliance and Certifications

- [ ] SOC 2 Type II report
- [ ] ISO 27001 certification
- [ ] Industry-specific compliance (PCI, HIPAA, etc.)
- [ ] Certification scope covers services
- [ ] Certifications current
- [ ] Right to audit clause

---

## Data Security

### Data Handling

- [ ] Data classification scheme
- [ ] Data handling procedures
- [ ] Data retention policy
- [ ] Data deletion procedures
- [ ] Data location documented
- [ ] Cross-border transfer mechanisms

### Data Protection

- [ ] Encryption at rest (algorithm, key management)
- [ ] Encryption in transit (TLS version)
- [ ] Data segregation (multi-tenant)
- [ ] Data masking/tokenization
- [ ] DLP controls
- [ ] Backup encryption

### Data Access

- [ ] Access control policy
- [ ] Least privilege enforced
- [ ] Access reviews conducted
- [ ] Privileged access management
- [ ] Customer data access logging

### Customer Access
